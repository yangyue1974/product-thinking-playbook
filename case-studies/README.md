# Case Studies

这 6 份文件是 `principles/` 里 20 条原则的**原始出处**。每一份都是一个已上线产品的全过程对话提炼，不是事后补的说明文档。

## 6 个产品

| 案例 | 产品形态 | 核心提炼出的原则 |
|---|---|---|
| [Contra Code](contra-code.md) | VS Code 内嵌魂斗罗游戏浮层 | 场景先于功能、愿景不妥协、借力认知、决策果断细节挑剔 |
| [AudioLib](audiolib.md) | 开发者音频库 API 服务 | 先定性再动手、Infrastructure 心智、气质即立场、开发者体验 |
| [Gospel Hub](gospel-hub.md) | 福音音乐人/发行/演出聚合站 | 指挥者姿态、不信任默认、审慎成本、基础设施节奏 |
| [Spybook](spybook.md) | 互动侦探叙事小说 | 安全=品质、端到端修复、核心指标绑定真实投入、计划先行 |
| [Tarot](tarot.md) | AI 塔罗咨询 | 产品边界、设计 vs 执行错误、持久化=信任、跨端一致 |
| [Xianxia](xianxia.md) | AI 仙侠小说生成 | 气质先于功能、内容标准、用户选择权、分发意识 |

## 怎么读这些案例

1. **顺着读**：按时间顺序，看产品思维是怎么在一个个项目里慢慢长出来的
2. **对照读**：先看某条原则，再来对应的案例看"当时是什么对话促成了这个判断"
3. **带着问题读**：你手头的产品卡在哪里？找最相似的案例对照
